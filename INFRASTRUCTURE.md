⚠️ **Important**: DNS is managed in **Cloudflare**, not Vercel. Vercel was the wrong place. Records added in Vercel for this domain are ignored.

### Cloudflare
- **Account**: _add email_
- **Dashboard**: https://dash.cloudflare.com
- **Active records (DNS only — proxy disabled, grey cloud)**:
  | Name | Type | Value | Notes |
  |---|---|---|---|
  | `n8n` | A | `5.78.151.169` | Points to Hetzner. Proxy = OFF (DNS only). |

### Why proxy must be OFF
If Cloudflare proxy is ON (orange cloud), Cloudflare intercepts traffic and breaks Coolify's Let's Encrypt SSL setup. Keep new records as DNS-only unless you specifically need Cloudflare's CDN/WAF.

- DNS records added inside Vercel will NOT take effect for greenbit.info

## Future apps roadmap

The same Hetzner server can host more apps via Coolify. To add another app:

1. Coolify dashboard → New Resource → pick a service or Docker image
2. Add a DNS record in Cloudflare: `<subdomain>.greenbit.info` A → `5.78.151.169` (proxy off)
3. Set the domain in Coolify to `https://<subdomain>.greenbit.info`
4. Coolify auto-issues SSL cert and routes traffic

Resource budget on CPX22: comfortably runs 2-3 modest services alongside n8n. If you outgrow it, upgrade to CPX32 (8 GB RAM, ~$16/mo) — one click in Hetzner console, no migration needed.

## When something breaks — debugging order

1. Is the domain reachable? `dig @1.1.1.1 n8n.greenbit.info +short` should return `5.78.151.169`.
2. Is the server reachable? `ping 5.78.151.169` from your machine.
3. Is Coolify alive? Open `http://5.78.151.169:8000`.
4. Is n8n container running? In Coolify → n8n service → status should be green "Running (healthy)".
5. Logs: Coolify → n8n service → Logs tab.
